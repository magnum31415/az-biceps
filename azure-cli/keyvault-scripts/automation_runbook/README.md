# Azure Key Vault Backup Automation - Architecture

## 1. Overview

This solution implements an automated backup mechanism for Azure Key Vault using Azure Automation.

The goal is to:
- Periodically back up Key Vault secrets, keys, and certificates
- Store backups securely in Azure Storage
- Use Managed Identity for secure, passwordless authentication
- Ensure idempotent and repeatable deployment via scripting



This diagram shows:

- **Deployment Plane**  
  `deploy-automation.sh` creates all resources

- **Execution Plane**  
  `Schedule → Runbook`

- **Identity Plane**  
  Managed Identity accesses Key Vault and Storage

- **Data Plane**  
  Backups → Blob Container

## 2. Architecture Components

### 2.1 Azure Automation Account
- Hosts the PowerShell runbook
- Executes the backup process on a schedule
- Uses System Assigned Managed Identity for authentication

### 2.2 Managed Identity
- Automatically created and assigned to the Automation Account
- Used to authenticate against:
  - Azure Key Vault
  - Azure Storage Account

### 2.3 Azure Key Vault
- Source of the backup
- Contains:
  - Secrets (excluding certificate-backed secrets)
  - Keys
  - Certificates

### 2.4 Azure Storage Account
- Destination of the backups
- Stores `.bak` files generated by Key Vault backup operations
- Organized using a structured path: ``<kvName>/<timestamp>/<artifact>.bak``

- Dry-run mode for validation

## 7. Configuration Management

Configuration is externalized via:

- `config.sh` → infrastructure parameters
- Automation Variables → runtime configuration

This ensures:
- Separation of concerns
- Reusability across environments
- No hardcoded values in runbook

---

## 8. Data Organization

Backups are stored using the following structure:

````
<keyVaultName>/<timestamp>/
├── secret-.bak
├── key-.bak
└── cert-*.bak
````

This allows:
- Easy restore operations
- Time-based versioning
- Logical grouping per execution

---

## 9. Reliability Considerations

- RBAC assignments include retry logic (eventual consistency)
- Runbook includes try/catch blocks for each operation
- Temporary files are always cleaned up
- Deployment script supports dry-run mode
- Verification steps ensure resources are correctly created

---

## 10. Limitations

- Azure Automation Variables require JSON workaround for string values
- RBAC propagation may introduce delays
- Storage configuration (versioning, soft delete, replication) is at account level, not container level

---

## 12. Summary

This solution provides a secure, automated, and scalable mechanism to back up Azure Key Vault using native Azure services, aligned with best practices:

- No secrets stored
- Fully automated
- Idempotent deployment
- Clear separation between platform and workload configuration
